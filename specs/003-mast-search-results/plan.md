# Implementation Plan: MAST Search Integration and Results Table

**Branch**: `003-mast-search-results` | **Date**: 2026-04-03 | **Spec**: [specs/003-mast-search-results/spec.md]
**Input**: Feature specification from `/specs/003-mast-search-results/spec.md`

## Summary

Integrate `astroquery.mast` to allow users to search for astronomical observations by object name. Results will be displayed in a responsive TUI table supporting fixed headers and 2D scrolling (vertical/horizontal). A startup welcome message will guide users, and a `/clear` command will reset the view.

## Technical Context

**Language/Version**: Python 3.12+  
**Primary Dependencies**: `blessed`, `astroquery`, `astropy`  
**Storage**: In-memory "Display State Buffer" (storing `astropy.table.Table` objects)  
**Testing**: `pytest`  
**Target Platform**: Terminal (TUI)
**Project Type**: CLI Application  
**Performance Goals**: Responsive UI during network fetches (<500ms for local UI updates)  
**Constraints**: Terminal size (min 80x24), fixed terminal state safety via `blessed`  
**Scale/Scope**: Prototype focusing on CAOM (Common Archive Observation Model) observations.

## Phase 2: Implementation Strategy

1. **Module: Archive (`src/mast_tui/archive.py`)**
   - Implement `MastClient` class with a `query_observations(object_name, radius="0.02 deg")` method.
   - Use `astroquery.mast.Observations`.

2. **Module: Table Component (`src/mast_tui/ui/table.py`)**
   - Implement `TableWidget` class to manage `scroll_x`, `scroll_y`, and rendering logic.
   - Handle fixed headers and 2D slicing of row strings.

3. **Module: Main Loop (`src/mast_tui/main.py`)**
   - Integrate `MastClient` and `TableWidget` into `AppState`.
   - Implement background threading for queries using `threading.Thread`.
   - Update `process_input` to handle arrow keys for scrolling when results are present.
   - Implement `/clear` command.
   - Implement welcome message logic.
